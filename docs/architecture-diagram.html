YES WMS - System Architecture

Warehouse Management System — High-Level Architecture Overview

Client Layer
🌐 Web App
React + Vite + TailwindCSS HTTPS
Main web interface for warehouse management. Firebase Auth for login. Communicates with backend REST API on port 5173 (dev).
📱 Mobile App
React Native + Expo HTTPS
Field operations app for warehouse staff. Handles picking, putaway, and scanning workflows on handheld devices.
📊 Ops Dashboard
Angular 17 + Material HTTPS
Operations monitoring dashboard. Real-time visibility into warehouse transactions, inventory levels, and worker activity.
🤖 Claude AI Agent
MCP Protocol SSE
AI agent bridge via Model Context Protocol. Claude can query inventory, create transactions, and manage warehouse data through MCP tools.
Gateway & Routing
🛡 Kubernetes Ingress
Reverse Proxy + TLS termination
Entry point for all traffic. Handles TLS, load balancing, and routes to the application server. Health probes at /api/v1/health.
🔀 ASGI Router
/api/v1/* → Django   /mcp → Starlette   /ws/* → WebSocket   /admin → Django Admin
ASGI application entry (wms_middleware/asgi.py). Routes traffic to Django Ninja API, MCP server (Starlette SSE), WebSocket handler, or Django Admin based on path prefix.
Middleware Pipeline
CORS
Cross-origin resource sharing. Allows frontend apps (ports 4200, 5173) to call the API. Credentials enabled, preflight cached 24h.
→
Request ID
Generates or propagates X-Request-ID header for distributed tracing across all log entries and downstream services.
→
Request Logging
Structured JSON logging of every request/response. Includes method, path, status, duration, org_id, facility_id. Sensitive fields auto-redacted.
→
Dual Auth
Primary: Firebase ID token verification. Fallback: X-API-Key header (when AUTH_FALLBACK_ENABLED=true). Attaches auth_source to request context.
→
Tenant Context
Resolves organization and facility from X-Org-Id and X-Facility-Id headers + warehouse key. Sets multi-tenant scope for the entire request.
Application Core — Django Ninja API (v1)
🏢 Masters
app.masters
Master data management: Organizations, Users, Facilities, SKUs, Zones, Locations, Roles, Permissions. Multi-tenant with org-scoped access control.
📦 Operations
app.operations
Warehouse transactions: GRN (Goods Receipt), Putaway, Move, Order Pick. Tracks Picks/Drops with full lifecycle (Pending → In Progress → Completed).
📋 Inventory
app.inventory
Real-time inventory visibility. InventoryBalance (on-hand, reserved, available) and InventoryLedger (full IN/OUT audit trail per transaction).
🧠 AI Chat
app.ai
AI-powered warehouse assistant. Conversations with tool-calling LLMs (Ollama, OpenAI, Claude). pgvector embeddings for semantic search over WMS data.
🔗 Connectors
app.connectors
External ERP/WMS integration framework. BaseConnector abstraction with Stockone provider. Syncs products, inventory, orders, suppliers, customers.
📄 Documents
app.documents
Transaction document generation. HTML/PDF creation per configurable templates. Uploads to Firebase Storage. Per-org/facility/type config rules.
🔔 Notifications
app.notifications
Real-time notifications via WebSocket (facility broadcasts, user-specific events) and Firebase Cloud Messaging (FCM) push notifications to mobile devices.
🔒 Auth
app.auth
Authentication & authorization. Firebase token verification, user sync on first login, role-based access, warehouse key validation per facility.
Infrastructure Layer
🐘 PostgreSQL 16
pgvector extension TCP
Primary data store for all domain models. pgvector extension enables 768-dim HNSW-indexed vector embeddings for AI semantic search. Multi-tenant with org-scoped indices.
⚡ Redis
Broker (db/0) + Results (db/1) TCP
In-memory data store. db/0 serves as Celery message broker for async task queuing. db/1 stores task execution results and status.
⚙ Celery Worker
Async task execution ASYNC
Processes async tasks: connector syncs, document generation, FCM push notifications. Consumes from Redis broker queue.
⏰ Celery Beat
Periodic scheduler CRON
Scheduled task dispatcher. Runs dispatch_due_connector_syncs every N minutes (configurable). Enqueues tasks to Redis for worker pickup.
External Services
🔥 Firebase
Auth + Storage + Realtime DB
Google Firebase platform. Auth for identity tokens, Storage for document uploads, Realtime DB for mobile sync. Service account key required for server-side verification.
☁ AWS Firehose
Structured log streaming
Amazon Kinesis Data Firehose. Streams structured JSON logs to S3/data lake for analytics. Configurable via LOG_DESTINATION=firehose. Falls back to stdout if unconfigured.
💬 LLM Providers
Ollama / OpenAI / Claude
Large language model backends for AI chat. Ollama for local inference, OpenAI (GPT-4o) and Anthropic Claude for cloud. Supports tool-calling for WMS data queries.
🔄 External ERPs
Stockone + future providers
External ERP/WMS systems connected via the Connectors framework. Stockone is the first provider. Syncs products, inventory, orders, suppliers, customers on a schedule.
